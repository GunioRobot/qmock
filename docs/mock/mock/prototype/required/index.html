    instance method Mock#required 

    Mock#required( num ) → Mock
    • num (Number) – Number of required parameters

    A mock object interface can accept parameters, which can be either required or optional. By default the number of required parameters is set by Mock.accepts to the length of the expected parameter list, but Mock.required can be used to override this in the case where some parameters are optional.

    It is used in conjunction with Mock.overload() to set parameter expectations on a mock object interface.

    Example

    // All parameters optional, overloading is set to true (by default)
Mock.method('foo').accepts('bar', 'baz').required(0);

    Verification of presentations to interface will allow 0, 1, or 2 parameters to be passed (though will still test type/value)

    Note: In the above example, if method overloading has been set to false then the verify phase would only allow presentations of ZERO parameters to the mock object interface.
